# 七点结构法参考（Phase 7 质控）

> Dan Wells 七点结构法。作为改编质量的「结构闸门」。

## 一、七个结构点

| 结构点 | 位置 | 内容要求 |
|--------|------|---------|
| Hook 钩子 | 前 10-15% | 让读者/观众产生「然后呢」 |
| Plot Turn 1 情节转折1 | ~15% | 给主角不能回头的原因 |
| Pinch 1 挤压点1 | ~25% | 反派/阻力亮一次拳 |
| Midpoint 中点 | 50% | 推翻主角和读者之前的认知 |
| Pinch 2 挤压点2 | ~75% | 把主角逼到墙角 |
| Plot Turn 2 情节转折2 | ~85% | 主角找到翻盘钥匙（非机械降神） |
| Resolution 结局 | 最后 10% | 回应物理层+情感层+意义层 |

## 二、与 novel-master 数据对照

- 卷级大纲（`outline/arcN-*.md`）通常已天然形成七点骨架 → 直接复用为分集节点
- NCG 节点（`ncg.json` 的 `tags`）→ 标注每集主结构点（如某集 tags 含「事件」→ 偏 Midpoint）
- 角色情绪弧（`state.json` 的 emotion_value 跨集变化）→ 验证情感层 Resolution 是否兑现

## 三、校验检查单

```
[ ] 每集能定位到七点中的至少 3 个结构点
[ ] 卷末集覆盖 Midpoint 或 Plot Turn 2（强结构点）
[ ] Plot Turn 2 的解决方案来自前文伏笔（非凭空）
[ ] Resolution 同时回应物理层（事件）与情感层（角色弧）
[ ] 无连续 3 集以上结构点缺失（"水集"）
```

## 四、常见失败模式

1. **机械降神**：Plot Turn 2 的解决方案前文无任何铺垫 → 对照 foreshadowing.json 修正
2. **钩子空转**：连续集只有钩子没有推进 → 检查 NCG 节点是否真的有因果推进
3. **情感断层**：Resolution 只解决事件不解决情感 → 对照角色 emotion_value 弧线补情感落点
