# 钩子设计参考（Phase 1）

> 钩子（Hook）= 让读者/观众产生「然后呢？」的心理冲动的机制。
> 本文件整合**钩子科学（半结果传播链）+ novel-master 伏笔系统**。

## 一、钩子类型（10 种）

| 类型 | 标签 | 机制 | novel-master 伏笔对应 |
|------|------|------|----------------------|
| 信息差钩子 | INFO | 角色已知，读者不知 | 轻伏笔（暗示1次） |
| 悬念钩子 | SUSP | 直接抛未解问题 | 轻/中伏笔 |
| 反转钩子 | TWIST | 颠覆既有认知 | 重伏笔（暗示3次+） |
| 关系钩子 | REL | 人物关系变化/秘密 | 中伏笔 |
| 命运钩子 | FATE | 重大选择/宿命 | 重伏笔 |
| 真相钩子 | TRUTH | 世界规则真相揭晓 | 中/重伏笔 |
| 情感钩子 | EMOT | 强情绪拉扯 | 角色情绪值极值 |
| 危机钩子 | CRIS | 倒计时/致命威胁 | conflicts.json 死亡/危机 |
| 谜题钩子 | PUZZ | 物理/逻辑谜题 | 设定层伏笔 |
| 欲望钩子 | DESR | 强烈目标驱动 | 角色核心追问 |

## 二、六组原始心理驱力（强度依据）

`SCARCITY` 时间稀缺 · `TERRITORY` 身份被侵犯 · `HUNT` 追逐快感 ·
`TRIBAL` 怕被排除 · `CONTROL` 信息不对称 · `LOSS` 即将失去

强度叠加：单驱力可靠；双驱力更凶；三驱力=核弹。每集按内容决定叠加层数。

## 三、半结果传播链（核心引擎）

每集结尾必须是「半结果」——只解答一部分，保留更大悬念。

五种半结果模式：
1. **答一半留一半**：解了小疑问，抛出更大的
2. **假答案**：看似解决，实为误导
3. **代价答案**：得到答案但付出沉重代价
4. **视角切换**：换视角揭示"原来如此"，但新的盲区出现
5. **时间锁**："三天后必须……"把悬念锁进倒计时

传播链模板：
```
集1: 钩子A → 半结果A(答一半) + 新钩子B
集2: 钩子B(承接A) → 半结果B(假答案) + 新钩子C
集3: 钩子C(承接B) → 半结果C(代价答案) + 大钩子D
...
```

## 四、钩子金字塔（层次结构）

```
全书大悬念（1-2，贯穿，大结局揭晓）
   └─ 卷级悬念（每卷 1-2，卷末解决或延续）
        └─ 批次钩子（每批次 1，推动）
             └─ 章节钩子（每章结尾，让人看下一章）
```

规则：
- 强/弱钩子交替出现
- 每 3-5 集一个强钩子集
- 季终集最强收尾

## 五、钩子道德边界（三条红线，自动对照）

| 检查项 | 问题 | 判定 |
|--------|------|------|
| 真实性 | 时限/数量限制在故事世界内是否真实成立？ | 成立 ✅ / 不成立 ❌ |
| 透明性 | 规则清晰不暗改，确定性承诺是否兑现？ | 兑现 ✅ / 暗改 ❌ |
| 可逆性 | 角色关键选择是否保留回头空间？ | 保留 ✅ / 绑架 ❌ |

## 六、与 novel-master 数据映射

- `tracker/foreshadowing.json` 的「暗示层次」→ 钩子强度（轻3/中4/重5）
- `tracker/foreshadowing.json` 的「回收状态」→ 钩子解答进度（未回收=持续钩子）
- `tracker/conflicts.json` 的「death/危机」→ 危机钩子 CRIS 落点
- `settings/characters/<名>/state.json` 的 `emotion_value` 极值 → 情感钩子 EMOT
- `tracker/plot_bus.json` 的 `foreshadow_trigger` → 钩子埋入集定位

## 七、强制输出格式（Phase 1 报告节选）

```markdown
## 钩子强度曲线（钩子金字塔）
全书: 第三块铁片（CONTROL+FATE，贯穿）
卷1: 铁片材质链（fp-002a~f，TRUTH）
批次: 邵峰之死余波（EMOT+CRIS）

## 钩子传播链（前3集）
第1集: fp-002a(半结果:知铁片存在不知用途) → fp-002b
第2集: 铁片交付(假答案:以为是护身符) → fp-002c(铁片=枪管材质)
第3集: 材质链(代价答案:得知真相但失去线索) → 远荒方向

## 驱力分布
卷1主驱力: CONTROL；第3集叠加 LOSS
```
