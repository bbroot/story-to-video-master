# 角色圣经 / 特征卡参考（双模式共用）

> 借鉴 `storyboard-sketch-narrative` 的「角色圣经」机制（跨镜头持续维护、更新即确认）
> + `novel-to-storyboard` 的「角色/场景特征卡提取」。

## 一、角色圣经（会话级，跨镜头持续维护）

### 1.1 注册格式（仅名称 + 角色音色，不记外观/空间描述）

```
=== 角色圣经 ===

【角色】
@林渡（音色：偏中性青年男声，分析派，句尾略带不确定升调）
@陆沉（音色：低沉克制，青年男声，情绪内敛）

【场景】
@工坊清晨
@灰岩堡训练场

【道具】
@第一块铁片
@弩臂
```

### 1.2 注册规则
- **novel-master 适配模式**：角色圣经直接由 `settings/characters/<名>/voice.md` 初始化
  （音色描述取 voice.md 的 speech_style 字段），场景/道具由章节正文 + geography.md 自动登记。
- **普通模式**：扫描用户粘贴的小说文本，自动提取所有角色/场景/道具，每个实体仅登记一个条目；
  角色额外自动设计一条音色描述（符合性别/年龄/剧情气质）。
- 已登记的实体**复用已有条目，不新增版本**；用户可随时说「添加角色」「重置角色圣经」。
- **更新即确认（硬规则）**：每次角色圣经变化（新增/改音色/重置）必须立即输出变更内容等待确认。

### 1.3 普通模式角色/场景特征卡（无 novel-master 数据时）

当无项目结构时，从文本自动提取特征卡，作为三视图锁定的替代来源：

```json
{
  "name": "林渡",
  "roleCardId": "char_lindu_v1",
  "gender": "男",
  "identity": "（从文本推断：如年轻教官/学徒）",
  "personality": {"core_traits": "（从行为推断）", "speech_style": "（推断音色）"},
  "appearance": {"description": "（首次出场描写）", "distinctive_features": ["（标志性特征）"]},
  "visual_stability_constraints": ["同一角色全程五官清晰稳定不变形"],
  "three_view": {"front":"待生成","side":"待生成","back":"待生成","status":"pending"}
}
```

提取规则（借鉴 novel-to-storyboard 的解析逻辑）：
- 角色：出现 ≥2 次且有人称/姓名/称谓的实体；首次描写句即外观锚点
- 场景：出现 ≥2 次且有空间/环境描写的名词短语
- 道具：被动作反复作用的物体（铁片/弩臂/信件）

## 二、与 novel-master 角色卡的关系

| 数据来源 | 角色圣经初始化方式 |
|---------|------------------|
| novel-master voice.md | 直接继承（含 v2 state 实时位置/情绪） |
| 普通文本 | 自动提取（见上） |
| 两者都无 | 用户口述补充，触发「更新即确认」 |

## 三、更新即确认输出模板

```
📖 本次角色圣经更新内容：
- 角色：@林渡（音色：偏中性青年男声，分析派，句尾略带不确定升调）
- 场景：@灰岩堡训练场
- 道具：@第一块铁片

无修改请回复「确认」；如需调整，请告知修改方向。
```
